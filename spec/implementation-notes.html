Melissa Map Implementation Notes

Design Decisions

  • V1 is static-first: no live geocoding or corpus-building calls during gameplay.
  • Primary mode is click-the-intersection; other modes are deferred.
  • CSS should be token-driven so a later aesthetic pass can reskin without rewriting game logic.
  • The map renderer is MapLibre GL JS behind the original small MapAdapter contract, so gameplay and scoring remain independent from the rendering library.
  • Question order is deterministic by seed and intersection id, which keeps QA repeatable while avoiding hardcoded array order as gameplay order.
  • Guess resolution uses haversine distance between lat/lon coordinates; screen position is only used to convert a click into a guess coordinate.
  • Scoring is intentionally coarse. A click is correct when the requested intersection is the nearest known intersection within the snap radius; correct guesses display at the snapped intersection, while misses name the nearest known intersection and receive 0-4 closeness points instead of 0-100 coordinate points. Guesses beyond 12 km receive zero.
  • Seed corpus lives in src/data/intersections.json; src/data/intersections.ts adapts it into the map/data Intersection contract.
  • Seed coordinates are playable approximate points, with uncertainty documented in src/data/seed-notes.md.
  • App shell uses React + Vite + TypeScript with small components and a tokenized CSS layer for the later aesthetic pass.
  • Game-loop state consumes the map/data workstream exports instead of duplicating question selection or scoring.
  • The current basemap uses OpenFreeMap's Bright vector style. MapLibre supplies native road hierarchy, label collision handling, and zoom-dependent detail without an API key or billing account.
  • Every text-bearing symbol layer is hidden while guessing and restored to its style-defined visibility after reveal. This prevents neighborhood, POI, airport, and road labels from leaking answer clues.
  • The former CARTO raster renderer and its client-side tile recoloring were removed rather than retained as a fallback.
  • The map default now focuses on the developed Las Vegas valley rather than the full Lake Mead / far desert extent, but starts slightly farther out than the first tuned pass so more valley context is visible.
  • MapLibre provides zoom, drag, touch, and wheel interaction; the adapter adds a valley-frame reset control and prevents rotation/pitch.
  • The starting valley frame is also the minimum zoom level for the current viewport.
  • Reveal mode preserves the current zoom and only nudges the center when the guess or answer would be too close to the viewport edge.
  • Starting a fresh unrevealed question restores the default valley frame so any reveal/inspection zoom does not carry forward.
  • Close marker labels split left/right to avoid stacking when the guess is nearly correct.
  • Reveal mode does not repeat the full intersection name on the map; the sidebar carries the prompt while the map keeps only answer/guess markers and native road context.
  • Wrong reveals do not render generated learning labels from the seed corpus. OpenFreeMap's native vector labels carry nearby street learning context.
  • The reveal-state instruction chip is a real Next button wired to the existing game-loop action, so players can advance without scrolling to the sidebar footer.
  • The reveal action uses the same route-red primary treatment as the sidebar Next action so its affordance stays consistent across the layout.
  • Local progress now persists and displays cumulative points by adding each guess's 0-5 closeness score. Existing saved progress safely starts this new field at zero.
  • Feedback labels use a lighter game voice: Nailed it, So close, In the neighborhood, Across town, and Lost in the valley. Road-specific feedback appears only when the road-axis heuristic actually identifies one or both answer roads.
  • The player-facing UI calls the two choices modes, not lessons. Setup, replay, miss-list, and summary copy avoids coaching language in favor of a relaxed road-trip voice.
  • The map-overlay and sidebar Next actions share the same Jost font, size, weight, tracking, route-red color, and paper text color.
  • The mode switch reads Find streets / Find intersections. The selected option uses route red, an inset border, a single right-aligned checkmark outside the text flow, and native aria-pressed state.
  • The public masthead and browser title read Learning Las Vegas; Melissa Map remains the internal project/repository name.
  • The About modal carries Brian's Melissa origin story plus a factual stack note covering React, TypeScript, Vite, MapLibre, OpenFreeMap, and the simplified OpenStreetMap geometry used for whole-road scoring.
  • Street mode keeps the curated street-geometry overlay hidden before a guess. The ordinary unlabeled basemap remains visible, while target and comparison geometry appears only after reveal.
  • Every exact four-point hit animates the shared reward card and +4 score, including street mode where there is no point marker. Intersection hits also animate the marker dot with a short stamp and expanding route-red ring. The marker container itself is not transformed because MapLibre owns its placement transform, and reduced-motion preferences disable all exact-hit animations.
  • Correct reveals use one enlarged “Nailed it” marker at the answer instead of comparing overlapping Guess and Correct pins. Misses retain separate markers for learning context.
  • Map attribution uses MapLibre's compact information toggle so required credits remain available without compressing into the map at narrow widths.
  • Intersection scoring treats seed rows within 150 meters as alternate names for one physical crossing. This prevents Charleston/Rampart from losing a nearest-point tie to the co-located Charleston/Fort Apache row while preserving the 650-meter answer radius.
  • A small masthead About link opens an accessible native modal. Its story copy is intentionally placeholder text for Brian's later wording.
  • The visual system now takes its cues from mid-century road atlases: warm paper, cartographic navy, route-marker red, muted green, geometric wayfinding type, condensed road lettering, fine rules, and squared paper-like controls. The treatment does not alter gameplay or map contracts.
  • The typography was pushed from a general heritage field-guide look toward a late-1950s road-atlas system: Jost supplies Futura-adjacent geometric display type, Barlow Condensed handles road and legend language, and Source Sans 3 remains the dense-reading face.
  • OpenFreeMap Bright is transformed before MapLibre commits the style: hosted glyphs are removed, symbol layers use locally rendered Jost or Barlow Condensed stacks, and existing source data, placement, collision, zoom hierarchy, and reveal visibility remain intact.
  • The desktop layout keeps the map dominant with a compact 390px field-guide sidebar; mobile stacks the map first and preserves large touch targets, visible focus states, and readable metadata.
  • Result feedback can call out when the guess appears close to one road but misses the cross street.
  • The Lake Mead Boulevard / Las Vegas Boulevard North seed point is explicitly guarded in validation after QA found it had drifted west to the I-15/Lake Mead interchange.
  • Area-specific play uses a normalized area bucket separate from the descriptive region field. This keeps the UI stable while preserving more granular seed notes.
  • The selected area is stored with local progress. Changing areas resets the current deck for that area; using Reset clears progress while preserving the selected area.
  • Coordinate QA now has two layers: dependency-free static guards in npm run validate:seed for known corrected rows, plus network-backed OSM road-crossing verification in npm run verify:seed:coords.
  • The Spring Mountain / Paradise prompt was corrected to Sands Avenue / Paradise Road because OSM road geometry shows the east-of-Strip crossing under the Sands Avenue name.
  • The invalid/non-crossing Flamingo / Lamb and Spring Mountain / Maryland rows were replaced with OSM-verifiable Flamingo / Pecos and Desert Inn / Maryland rows.

Deviations

  • The road-read feedback is heuristic. It infers street orientation from the seed corpus and should be reviewed as more intersections are added.

Tradeoffs

  • Manual seed corpus is faster and safer for V1 than building the full OSM/Clark County ETL immediately.
  • OpenFreeMap is intentionally used without a CARTO fallback because service reliability is not critical for this version.
  • The sample corpus remains available for tiny fixtures, but app gameplay should use the 76-item intersections export.
  • The first corpus started at 70 intersections, grew to 76 to give every exposed area bucket at least 10 prompts, and then to 130 to reduce repetition and cover more combinations of the existing streets.
  • The basemap now depends on public OpenFreeMap style, vector-tile, glyph, and sprite availability. Self-hosted tiles or PMTiles remain future options if that dependency becomes unacceptable.
  • Reveal no longer zooms in to expose labels because that removed too much location context; split labels and a small preserve-zoom pan are the current compromise.
  • Removing the answer-street callout makes the map less self-contained, but avoids duplicating the prominent prompt that remains visible in the sidebar.
  • Native vector labels are geographically faithful and gain detail as the player zooms, but their density and typography remain governed by the external OpenFreeMap Bright style.
  • Spot-check coordinate guards are intentionally narrow for now; broader corpus precision still needs a future ETL or review pass.
  • The static build is deployed to the permanent here.now site at https://clever-grove-3t69.here.now/. The OpenFreeMap Bright release was browser-verified through guess, reveal, map-overlay Next, and reset-to-unrevealed flow.
  • The road-atlas visual release was deployed to the same permanent here.now site after responsive browser review. The production guess, reveal, and map-overlay Next path was re-verified after publish.
  • The later geometric/condensed typography refinement was deployed to that same site after confirming Jost, Barlow Condensed, and Source Sans 3 loaded and that transformed native labels remained readable on desktop and mobile.
  • The 0-5 scoring and success-marker release was deployed and production-verified with representative far and exact clicks. The far case preserved a zero running-total increment; the exact case rendered one enlarged Nailed it marker.
  • Playable area buckets require at least 10 intersections before being shown. Henderson/Green Valley and North Las Vegas/Northwest are being topped up manually rather than splitting them into smaller sparse regions.
  • The first area chooser is a native select in the sidebar instead of a full start screen; this preserves the fast first prompt while still letting the player choose All Areas or a specific bucket before guessing.
  • The OSM verifier is intentionally separate from the normal build because it depends on public Overpass availability and live map data.
  • Rows that do not produce an OSM road crossing should be treated as content QA failures, not just coordinate misses; the verifier exposes those instead of hiding them behind approximate points.
  • The game uses two explicit modes. Both normal session types draw 10 distinct items and rotate coverage across sessions; only the explicit Replay the misses action seeds later repetition.
  • Every learning session is exactly 10 prompts and 40 possible points. Both street and intersection scoring use the shared 0-4 distance bands; four points requires resolving to the requested eligible item within the 650 m snap radius.
  • Street mode is grouped internally by comparable geography: three ordered grid groups and one special-shape group containing Rampart, Spring Mountain, Paradise, Green Valley Parkway, and Boulder Highway. Shape streets compare against the whole shape group without claiming a false cardinal ordering.
  • Progress remains browser-local under melissa-map-progress-v2. It stores mode selection, active-session recovery, completed-session count, best score, and the last session's misses; the old selected area is migrated from the V1 key.
  • Coarse pointers stage a map coordinate before scoring. The map instruction square then changes in place to a route-red “Confirm guess?” button, avoiding the former overlapping second control. Its Jost typography matches the map's Next action.
  • The MapLibre adapter owns the street GeoJSON source and line layers. Updates before the external style is ready are retained in adapter state and rendered from the style.load boundary instead of touching an uninitialized style.
  • The visible eligible-street button list and pre-session five-road briefing were removed after playtesting. The list allowed the requested answer to be selected directly for full points, while the five-road repeat loop made the mode too constrained to teach the broader valley.
  • The Henderson-focused street expansion adds Stephanie Street, St. Rose Parkway, Horizon Ridge Parkway, and Lake Mead Parkway as fully playable regional shapes, raising the corpus from 28 to 32 roads.
  • The intersection corpus now contains 130 OSM-verified crossings, including 54 new combinations drawn only from the existing 32-road street corpus. Area pools range from 18 to 24 prompts.
  • Normal intersection rounds now contain 10 distinct crossings. Automatic weakest-item repeats were removed; Replay the misses remains the explicit repetition path.
  • Each submitted guess stores elapsed milliseconds from prompt display to submission. The round summary shows total and average answer time without adding countdown pressure during play.
  • Street-mode nearest-guess geometry uses a continuous dark-slate line instead of a dash array. The narrower width and explicit label keep it distinct from the solid route-red answer while avoiding segment-phase breaks across zoom levels.
  • Street reveals support a separate static roadside-note layer for memorable landmarks, navigation quirks, corridor identity, history, and occasional name origins. The 12-street pilot keeps these sourced editorial facts distinct from the existing spatial teachingNote.
  • Roadside-note provenance ships with the local data and is exposed as a compact source link. Streets outside the pilot render no empty note container, and the app makes no runtime content request.
  • The second roadside-note pass expands coverage from 12 to 24 of 32 playable streets. Eight roads remain uncovered because available material was too generic or weakly sourced; coverage is not a completeness metric worth lowering the editorial bar for.

Open Questions

  • Future QA should tune the snap radius as the corpus grows denser, especially in areas with closely spaced intersections.
